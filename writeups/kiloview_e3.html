Unauthenticated Factory Reset Vulnerability in Kiloview E3 Video Encoder

Confrmed for Software Version 1.20.0006

Summary

An unauthenticated API endpoint on Kiloview E3 video encoders allowed remote attackers to trigger a factory reset without any credentials or prior authentication. If the device remained reachable after the reset, it was accessible via default credentials, allowing full dashboard and video feed access.

Device Discovery

While performing passive recon, I located a Kilview E3 encoder with a publicly exposed login portal:

Reconnaissance of Web Interface

Inspecting hte login page's source revealed a linked JavaScript file which most likely was responsible for the web application logic

Application Logic and API Discovery

The JavaScript was viewable within a web browser by going to the URL path associated with it. This led me to discover that the web application called API functions through specific URL addresses. To call an API in the URL, the URL is prefixed with a value called "proxyUrl" which is defined as the machine's IP address followed by /api/.

Identification of Vulnerable Endpoints

Following this API URL definition were a series of API functions with their respective URLs to call them. Some functions had parameter values that were handled in thier execution. However functions with no parameters could be called without any authorization, which including the factory reset function "systemctrl/system/reFactory".

Exploitation

After crafting the URL payload to call the factory reset, I entered it into the URL bar which returned a screen that confirmed the execution of the function.

Post Explotation

After factory reseting the device, I have observed two possible outcomes.

The most common outcome is the device is no longer reachable post factory reset. Assuming that the reset erases a specific configuration or setting that had intially made it publicly accessible.

The other outcome being that the device is still publically accessible, which allows the attacker to use the default credentials to login. Giving the attacker full adminstrative access to the device.
